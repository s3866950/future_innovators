Tools

Upon creating our group, for this and future assignments, we opened a discussion thread on Canvas about the best and most effective ways to move forward with communication. We all came to an agreement that Microsoft Teams was the ideal choice of software as our primary method of communication. The ease of communicating ideas via text, video, and calls on desktop or mobile would be very straightforward, and we would also have the means to share and edit files.

Tools used for this assignment

  • Canvas – We mainly utilised Canvas as a source for preliminary discussions, but as we progressed with the assignment we began using it more to make announcements as it was a more direct, visible way of voicing something urgent for those that hadn’t been checking Teams.
  • MS Teams – Microsoft Teams was our primary communication tool throughout this assignment. We were able to communicate and share files/input on the message boards that we created and began to develop good working relationships with those who utilised the tool properly. We also held team video calls where we were able to more comprehensively communicate points and ideas, a benefit that messaging does not provide.
  • Trello – The group did not start using Trello until a couple of weeks into the assignment, however when we realised how good of a tool it was for organising, uploading and viewing the assignment content it became an extremely useful asset. It provided us with a more in-depth scheduling device and a better visual representation of what needed to be accomplished.
  • GitHub – GitHub was a very useful tool to be able to visually identify who was contributing content and who was not. Although we did utilise Trello as a place to upload our files, we continued accessing GitHub as we were still unaware of any problems we may encounter using a new system. In a way it was a good backup for us should any of our content have been misplaced or lost throughout the assignment process.

Team Git Repository: https://github.com/shaggy982/rmitita2g9

As is always the case with GitHub, the version control aspect of it is extremely useful, although we did not need to make use of it this time around.

Although the GitHub repository was utilised to organise and upload files and information, the audit only vaguely reflects the work that was done. As it was used as a storage and organisational tool, it does not completely reflect the discourse and file sharing that took part during this project.

Our ability to employ the benefits of other tools such as Teams and Trello made the use of, and reliance on, the GitHub repository minimal. Those contributing to the assignment did make use of it; however, the contribution levels merely show the content added and do not reflect the exchange of ideas about said content.
